Introduction

Geany est un éditeur de texte multi-plateforme écrit en GTK+ et basé sur la librairie Scintilla. Il est particulièrement léger et comporte de nombreuses fonctionnalités facilitant le programmeur. Il a été conçu pour avoir un nombre réduit de dépendances permettant un temps de chargement rapide. Geany est disponible sur un grand nombre de système d'exploitation comme Windows, Linux, Mac OS X, BSD et Solaris. Il supporte de nombreux langages de programmation tel que : C, C++, Java, JavaScript, PHP, HTML, CSS, Python, Perl, Ruby, Pascal, Haskell ... Il est disponible sous la licence GNU GPL v2. site officiel : http://www.geany.org/

Détail

Approfondissement des fonctionnalités

L'auto-complétion

Le point le plus intéressant de Geany est sans doute son auto-complétion. Dans un langage de programmation impératif, il suffit de taper les 4 premiers caractères (par défaut) d'une de ses fonctions pour voir apparaitre un menu contextuel présentant la liste des fonctions ayant le même début de nom : Il suffit ainsi de sélectionner sa fonction dans la liste avec le clavier ou la souris. A l'ouverture de la parenthèse de l'appel de fonction, un petit cadre s'ouvre pour rappeler le prototype de la fonction. S'il existe plusieurs fonctions avec le même nom, il est possible de changer le prototype du cadre pour voir les autres prototypes. On change de prototype au moyen de deux petites flèches dans le cadre : Dans un langage de balisage, l'ouverture d'une balise va créer une fermeture à coté. Il existe même des complétions plus évolués comme la balise en HTML qui va générer automatiquement ce code : Encore plus d'auto-complétion sont disponibles dans Geany notamment au moyen des snippets. Les snippets sont un système d'alias de mots clés utilisant un fichier de configuration. Ainsi, par exemple en C++, taper for puis Tabulation donne le code for (int i = 0; i < ; i++) {} et place le curseur après le i < .

Support de nombreux langages

Geany supporte de nombreux langages de programmation. Cela signifie que la coloration syntaxique, l'auto-complétion et encore beaucoup d'autres outils sont disponibles pour ces langages. Ainsi sont supportés les langages : ActionScript, Ada, ASM, C, C#, C++, CAML, CMake, Conf, CSS, D, Diff, Docbook, F77, Ferite, Fortran, FreeBasic, GLSL, Haskell, Haxe, HTML, Java, Javascript, LaTeX, Lua, Make, Matlab, None, NSIS, Pascal, Perl, PHP, Po, Python, R, reStructuredText, Ruby, Sh, SQL, Tcl, Vala, VHDL, XML, YAML.

Coloration syntaxique

La coloration syntaxique du code est aussi très importante et maintenant systématique pour tout éditeur de texte de programmation. Il désigne la coloration des mots clés, des symboles et de tout les éléments du langages de programmation. Il a pour but de faciliter la lecture d'un programmeur et de mieux déceler des erreurs de programmation.

Pliage du code

Il est possible de plier le code c'est à dire de réduire un ensemble de code en une seule ligne pour le déplier par la suite. Il existe plusieurs ensembles de code pliables : les commentaires de code, les blocs, les balises, ...

Très paramétrable

Geany est un logiciel très paramétrable. D'une part, il existe de nombreuses options dans le menu Editer/Préférences permettant de modifier de nombreux paramètres tels que les raccourcis claviers. D'autres part, il est possible de modifier plus en profondeur certaines fichiers de configurations accessibles dans Outils/Fichiers de configuration.

Assistance de compilation, suivi des erreurs et execution du code

Nous allons expliquer cette fonctionnalité au moyen d'un programme en C. Les boutons suivants permettent respectivement de compiler et d'exécuter le code. Pour réaliser ces tâches, il faut préalablement définir les commandes associées dans le menu Construire/Définir les includes et les options. Pour cet exemple : considérons le code de l'exemple : #include #include // retourne le plus grand entier du tableau int tab_max (int tab[], int size) { int max=tab[0]; int i=1; while(i max ) max=tab[i]; ++ i; } return max; } int main (void) { int t[] = {5,-5, 4, 65, 6, 8, 4}; printf("%d\n", tab_max(t,7) ); return EXIT_SUCCESS; } Un clic sur Compiler m'indique Compilation terminée avec succès. dans la fenêtre des messages. Je peux si je veux Exécuter mon code qui me lance un terminal m'affichant 65. Maintenant, supposons qu'on oubliait de mettre l'inclusion #include dans le code. Voici le résultat de la compilation : Si je clique sur le message main.c:20: erreur: ‘EXIT_SUCCESS’ undeclared (first use in this function), Geany amène directement à la ligne de code de l'erreur et ceci même avec plusieurs fichiers. Cette gestion performante des erreurs permet une plus grande rapidité de développement. La possibilité de choisir la commande de compilation et d'exécution rend cette fonctionnalité excellente : on pourra par exemple compiler avec make ou encore exécuter avec valgrind pour tester des fuites de mémoire. Note: en plus de l'action "Compiler", il existe l'action analogue "Construire".

Sélection aisée d'une couleur hexadécimal

Il est possible de sélectionner une couleur d'une manière graphique et d'obtenir son code hexadécimal. Cette fonctionnalité est notamment très utile pour le CSS. La validation de cette fenêtre insèrera #0066CC à la position du curseur.

Plugins

Geany dispose de nombreux plugins. Je vous présente ceux que j'ai retenu.

Caractères HTML

Caractères HTML permet d'insérer facilement des caractères spéciaux depuis une liste ou d'en taper directement. En effet, il est possible d'écrire en mode HTML : tous les caractères spéciaux tapés seront automatiquement remplacés par la correspondance html (par exemple é pour é). Encore plus intéressant, ce plugin intègre un convertisseur texte vers html (remplacement en masse des caractères spéciaux dans le menu Outils / Remplacement HTML).

Debugger

Le plugin Debugger est une interface GDB pour le débugguage d'applications. Il est presque entièrement intégré à Geany. Avant de l'utiliser, n'oubliez pas l'option -g à la compilation sous GCC afin d'indiquer le mode debug. Il n'est pas possible de placer les breakpoint "à la souris" il faut entrer au clavier le numéro de la ligne où placer le breakpoint. Néanmoins, les champs pré-remplis concernant le fichier et le numéro de ligne sont récupérés à partir de la position du curseur. En résumé, pour poser un breakpoint, placez votre curseur à la position du breakpoint, cliquez sur Breaks puis validez. Pour débugger un programme, Il faudra charger manuellement le fichier exécutable même s'il a déjà été indiqué dans les options de la compilation intégrée. Utilisez le bouton Load. Diviser la fenêtre

Le plugin Diviser la fenêtre permet de splitter l'espace de travail Geany en deux afin de travailler sur un code source en s'aidant d'un second. Ce besoin est très fréquent. Ainsi le plugin évite une intermittence régulière entre deux morceaux de codes source.

Navigateur de fichier

Ce plugin permet d'avoir un petit navigateur de fichier intégré dans Geany depuis la barre latérale.

Création de Classe

Création de Classe permet de créer simplement une classe C++ ou une classe GTK+ en pré-remplissant quelques morceaux de codes. Ce plugin n'est pas encore très abouti mais il peut devenir très intéressant par la suite si son développement prend de l'ampleur.

Conclusion

Il est toujours plus agréable de programmer avec un éditeur de texte prévoyant la coloration syntaxique, l'auto-complétion et diverses autres fonctionnalités. De nombreux éditeurs IDE sont très aboutis mais parfois au détriment de la légèreté du logiciel. Geany se présente comme un logiciel efficace qui a sût trouver un équilibre entre légèreté et fonctionnalité.
